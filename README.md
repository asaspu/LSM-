LSM-KVStore 🗄️
基于 LSM-Tree 的轻量级 C++ 持久化 KV 存储引擎，参考 LevelDB 核心思想实现，无第三方依赖，适合学习 LSM 底层原理。

📖 项目介绍
本项目使用纯 C++ 从零实现极简 KV 存储，完整复刻 LSM 树分层架构，区分内存层与磁盘持久层，通过 WAL 预写日志实现宕机数据恢复，设计两级 Compaction 机制缓解读放大。
代码模块化拆分，逻辑清晰，剔除工业级复杂冗余特性，专注 LSM 核心运行流程，便于初学者理解存储引擎基础设计。

🔄 数据流转流程
写入流程
Put/Delete → 写入 WAL 日志持久化 → 存入内存跳表 MemTable
内存数据达到 4MB 阈值，触发 Minor Compaction，将内存数据落盘生成 L0 层 SSTable；
当 L0 文件数量到达阈值，触发 Major Compaction，合并 L0、L1 层文件，覆盖重复键。
查询流程
Get优先读取最新内存 MemTable；内存未命中时，从新到旧逆序遍历 L0 磁盘文件；仍未匹配再读取 L1 老旧文件。
宕机恢复流程
引擎启动时自动读取 WAL 日志，重放全部写入、删除操作，重建内存数据，保证异常断电数据不丢失。

⚙️ 核心功能
🧠 自研线程安全跳表 MemTable
底层手写跳表实现有序内存存储，增删查时间复杂度 O (logN)；内置互斥锁保证单表线程安全，支持导出全量有序数据用于落盘。

📜 WAL 预写日志
自定义二进制日志存储格式，区分写入、删除两类操作记录；每次修改先刷入磁盘日志再更新内存；内存数据成功持久化后自动清空冗余日志。

💾 SSTable 磁盘有序文件
不可修改的磁盘存储文件，每个文件记录全局最大、最小 Key，查询时可快速跳过无关文件；支持内存数据批量序列化写入磁盘。
🔧 两级 Compaction 合并机制
小合并 (Minor)：内存数据超限自动刷盘，释放内存空间；
大合并 (Major)：合并新旧两层磁盘文件，消除重复过期键，减少磁盘文件数量，降低查询 IO 开销。

🧪 内置全套测试用例
程序内置三组自动化测试：基础增删改查功能测试、十万级数据批量读写性能测试、宕机重启数据恢复测试。

🚀 编译 & 运行

📋 环境要求
Linux /macOS，支持 C++17 标准的 gcc、clang 编译器，无外部依赖库

🔨 编译命令
g++ main.cpp kvstore.cpp memtable.cpp skiplist.cpp wal.cpp sstable.cpp compaction.cpp -o lsm_kv -std=c++17 -lpthread

▶️ 运行程序
./lsm_kv

▶️ 额外配备了崩溃后WAL恢复测试
先./test.write    写入100条数据后崩溃

再./test.read     恢复这100条数据

⚠️ 实现简化说明
为降低学习门槛，本项目对工业级存储做了轻量化简化，仅保留核心流程：
SSTable 完整磁盘读取、Compaction 多路归并仅预留代码框架，仅保证基础流程可跑；
顶层 KVStore 未实现全局读写锁，单线程测试完全稳定；
未实现布隆过滤器、删除墓碑、数据 CRC 校验、LRU 缓存、多层 Level 等进阶优化。

✨ 后续可拓展优化方向
新增布隆过滤器，过滤不存在的 Key，减少无效磁盘 IO；
引入 Tombstone 墓碑机制，合并时彻底清理已删除数据，解决磁盘空间膨胀；
增加 LRU 块缓存，缓存热点磁盘数据，提升读取性能；
增加文件 CRC 校验，识别损坏的日志与 SST 文件；
顶层增加读写分离锁，支持多线程并发读写；
拓展多级分层 Level 结构，精细化管控每层文件数量；
实现迭代器、范围查询、批量写入等高级接口。
